一、需求文档

1.1 项目背景

在批量人像照片处理场景中（如活动摄影、证件照批量处理、粉丝见面会照片等），需要对大量图片中的特定人员进行美颜处理（主要是磨皮），而图像中的其他人不做处理。需要一套自动化流程实现"指定人员检测 → 选择性美颜"。

1.2 用户故事

ID角色用户故事优先级
US-01运营人员 上传一批活动照片，指定需要美颜的人员（提供该人员的正脸照片作为参考），系统自动找出包含该人员的照片并对其面部做磨皮处理 P0
US-02运营人员 支持同时指定多个人员（如团队照中多位核心成员），仅对他们做美颜，其他人保持原样 P0
US-03运营人员 可调节美颜强度（轻度/中度/重度磨皮），适应不同场景 P1
US-04运营人员 批量处理完成后，可预览每张图片的处理前后对比 P1
US-05运营人员 支持将处理后的图片批量导出，保持原始目录结构 P1
US-06运营人员 提供处理报告（处理成功数、失败数、未检测到目标的图片列表） P2

1.3 功能列表

模块功能说明优先级
输入目标人员注册上传 1~N 张目标人员的正脸照片作为识别基准P0
输入批量图片导入支持文件夹批量导入，支持 jpg/png/webpP0
核心人脸检测检测所有图片中的人脸位置P0
核心人脸识别将检测到的人脸与目标人员比对，确认身份P0
核心人脸分割生成目标人员面部区域的精确 Mask（排除头发/耳朵/背景）P0
核心磨皮处理对 Mask 区域应用磨皮算法P0
配置磨皮强度可配置磨皮等级（参数预设或自定义）P1
输出批量导出导出处理后的图片，保留原文件名和目录P1
输出对比预览生成处理前后对比图P1
输出处理报告输出 JSON/CSV 格式的处理报告P2
扩展GPU 加速支持 CUDA 加速批量处理P2

1.4 核心流程

1 注册目标人员（提供参考照片）

2 提取目标人员 Face Embedding

3 遍历输入图片，检测所有图片中的人脸

4 将检测到的人脸 Embedding 与目标 Embedding 比对，筛选出目标人员

5 对目标人员的人脸区域生成皮肤 Mask（排除五官边界区域）

6 对 Mask 区域执行磨皮算法

7 输出处理后的图片 + 处理报告

二、策划文档

2.1 产品定位

一款命令行 + Web UI 的批量人像美颜工具，核心差异化：指定人员选择性美颜而非全图无差别处理。

2.2 目标用户

  • 活动 / 会议摄影师，需要批量精修主办方人员
  • 粉丝站 / 后援会运营，处理大量活动图
  • 企业内部宣传照片批量处理
  • 证件照 / 肖像照批量精修

2.3 非功能性需求

维度要求
性能100 张图（含 1~5 个人脸/张）批量处理 ≤ 5 分钟（GPU）
精度目标人员识别准确率 ≥ 95%（正脸参考照条件下）
健壮性单人脸识别失败时不应中断整个批次
扩展性支持后续增加美颜类型（美白、红润、去瑕疵等）
隐私所有处理本地完成，不上传任何图片

2.4 MVP 范围

第一阶段（MVP） — 命令行工具，涵盖核心链路：

  • 目标人员注册（提供参考照片目录）
  • 批量图片人脸检测 + 识别
  • 选择性皮肤磨皮
  • 批量导出 + 简单报告

第二阶段 — 增加 Web UI、多美颜类型、对比预览、GPU 加速

第三阶段 — 视频帧处理、实时流处理、API 服务化

2.5 交付物

  • Python CLI 工具（含完整 Source Code）
  • 依赖清单 requirements.txt
  • README 使用文档 + 示例

2.7 页面详述

2.7.1 仪表盘（Dashboard）

维度说明
页面目的提供全局概览，快速发起批量处理或查看近期结果
入口位置应用首页，导航栏第一个标签

核心功能：

  1. 统计卡片 — 已注册目标人数、累计处理图片数、本周处理量、最后处理时间
  2. 最近任务列表 — 展示最近的 5~8 次批量处理任务（时间、目标人员、图片数、处理状态）
  3. 快捷操作区 — "新建批量处理"按钮（主按钮）、"注册新人员"辅按钮
  4. 快速拖入区 — 支持拖拽图片/文件夹到仪表盘直接进入批量处理流程

UI 元素清单：

元素类型交互要求
统计卡片卡片 × 4数字实时更新，鼠标悬停可有微动效；点击可跳转到对应详情页
最近任务列表列表行每行显示任务名、时间、状态标签（进行中/完成/失败）、缩略图预览；点击行跳转报告页
"新建批量处理"按钮Primary Button全局最醒目的主按钮（蓝色/渐变色）；快捷键 Ctrl+N
拖入区域虚线框 Dropzone拖入时高亮并显示 + 号动画；支持多文件和文件夹；松开后直接进入批量处理页并自动填充导入区
空状态占位插画首次使用时显示引导插画 + 文字："开始你的第一次批量处理"，下方配"导入图片"和"注册人员"两个入口按钮

状态与交互：

空状态： 用户首次进入时，无任务记录。显示引导插画 + "开始你的第一次批量处理" 文案 + 两个入口按钮。不显示空白表格。
加载态： 统计卡片显示骨架屏（Skeleton Loading），列表显示 4 行灰色占位条。
错误态： 数据加载失败时显示重试按钮，不阻断其他模块。

2.7.2 目标人员管理（Target Person Management）

维度说明
页面目的注册、查看、编辑、删除需要被美颜的目标人员
核心操作流添加人员 → 上传参考照片 → 系统自动提取 Face Embedding → 保存

核心功能：

  1. 人员列表 — 展示所有已注册目标人员，支持搜索和排序
  2. 添加人员 — 输入姓名 + 上传 1~N 张正脸参考照片
  3. 查看详情 — 展示人员基本信息 + 参考照片缩略图 + 注册时间 + 被处理次数
  4. 编辑人员 — 修改姓名、增删参考照片
  5. 删除人员 — 删除后该人员不再被识别（软删除，关联的历史任务保留）
  6. 试识别 — 上传一张测试照片，系统立即返回是否匹配该人员及置信度

UI 元素清单：

元素类型交互要求
人员卡片网格卡片 Grid每张卡片包含头像（参考照第一张）、姓名、照片数、最后使用时间；鼠标悬停放大阴影效果
搜索框Input + 图标支持实时搜索（输入即过滤，无需回车）；支持拼音模糊搜索
"添加人员"按钮Primary Button打开侧边滑出面板（Drawer），非弹窗，保持上下文
添加人员表单（Drawer）表单字段：姓名（必填）、备注（选填）、参考照片上传区（必填，支持多张）
照片上传区Dropzone + 缩略图预览支持点击选择或拖拽上传；上传后立即显示缩略图；每张可单独删除；至少 1 张且分辨率 ≥ 200×200
人员详情弹窗Modal"编辑"和"删除"按钮在底部操作栏；编辑时表单可修改；删除需二次确认（"确认删除 张三？此操作不可撤销"）
"试识别"按钮Ghost Button在人员详情或列表中均可操作；上传测试图→显示匹配结果和置信度百分比
人员头像Avatar 组件支持显示参考照第一张；无照片时显示姓名首字母圆形占位

状态与交互：

空状态： 无已注册人员时，显示插画 + "还没有目标人员，添加第一个人吧" + 大号"添加"按钮。
上传中： 照片上传时显示进度条/旋转加载；全部上传完成自动关闭加载态。
识别失败： 上传的照片中未检测到人脸时，Toast 提示"未检测到人脸，请更换照片"，不允许提交。
试识别结果： 匹配成功 → 绿色标签 + "匹配成功，置信度 96.3%"；不匹配 → 橙色标签 + "未匹配到目标人员"。

2.7.3 批量处理工作台（Batch Processing）

维度说明
页面目的配置并执行批量处理任务，是整个应用的核心工作台
页面结构左面板（配置区）+ 右面板（预览区）的二分栏布局

核心功能：

  1. 导入图片 — 选择文件夹或拖拽图片到导入区，展示文件列表（文件名、大小、预览缩略图）
  2. 选择目标人员 — 从已注册目标人员中选择一个或多个，支持全选/反选
  3. 配置美颜参数 — 磨皮强度滑杆、预设模式（轻度/中度/重度/自定义）
  4. 执行处理 — "开始处理" 主按钮，执行时显示实时进度
  5. 实时进度 — 进度条 + 当前处理图片缩略图 + 处理状态文字（"第 23/100 张"）
  6. 中途暂停/取消 — 处理过程中可暂停（Pause）或取消（Cancel），已处理部分保留

UI 元素清单：

区域元素交互要求
左面板
配置区		图片导入区（Dropzone）支持点击选择文件夹或拖拽；拖入时高亮 + "+" 动效；接受 jpg/png/webp；重复文件自动跳过并 Toast 提示
导入文件列表紧凑列表：缩略图(48px) + 文件名 + 文件大小 + 删除按钮；支持批量清空；统计数字（"共 42 张图片"）
目标人员选择器带搜索的多选下拉组件（Select+Multiple），选项显示已注册人员名 + 头像缩略图；已选人员以 Tag 形式展示在下方
参数配置面板滑杆(Slider) 控制磨皮强度（0~100）+ 预设模式快速切换按钮组（轻度/中度/重度）；高级选项折叠面板：保护边缘强度、保留细节比例
右面板
预览区		缩略图画廊以网格展示导入图片的缩略图；每张右下角标识"已检测到人脸数"；无检测到时显示灰色 "😶" 图标
单图放大预览（可选）点击缩略图放大查看；支持滚轮缩放；如已处理过则显示处理前/后切换
进度指示器（处理后）环形进度图 + "处理中" / "已完成" / "失败" 标签
底部
操作栏		"开始处理" 主按钮蓝色主导按钮；禁用条件（未导入图片 或 未选择目标人员）；快捷键 Ctrl+Enter
"重置" 按钮Ghost Button，清空当前配置但保留历史

状态与交互：

空状态： 左面板显示"拖拽图片到此处或点击选择文件夹"；右面板显示应用 Logo/引导图。
处理中状态（关键状态）：
  • 进度条 + 实时百分比（"58/100 张"）
  • 当前正在处理的图片缩略图在进度条上方闪烁
  • 已处理的图片在右侧画廊左下角显示绿色 ✓ 标记
  • 失败的图片显示红色 ✗ 标记，鼠标悬停显示失败原因
  • "开始处理"按钮变为 "暂停"（橙色）和 "取消"（红色）两个按钮
处理完成： 进度条变为绿色 100%；"查看报告" 按钮出现；右下角弹出系统通知（如支持）。
注意事项（防误操作）：
  • 处理中不允许导入新图片或更改目标人员选择
  • 暂停后可以调整参数并"继续处理"（从暂停处恢复）
  • 页面刷新后未完成的任务状态应保留（IndexedDB/localStorage）
  • 导入大量图片（>500 张）时弹出确认框："导入大量图片可能影响处理速度，是否继续？"

2.7.4 预览审阅（Review）

维度说明
页面目的批量对比处理前后效果，逐张确认或标记需重处理
进入方式批量处理完成后自动跳转，或从报告页/任务列表进入

核心功能：

  1. 对比查看 — 左右对照（原图 / 处理后）或滑条拖动对比
  2. 逐张审阅 — 键盘方向键翻页浏览所有图片
  3. 标记重处理 — 对不满意的图片打标记，后续可批量重处理
  4. 全屏模式 — 进入全屏以获得更大对比视野
  5. 批量导出 — 确认后导出到指定目录

UI 元素清单：

元素交互要求
对比视图默认左右并排（原图 | 处理后）；支持切换为"滑条模式"— 拖动中间分割线左右滑动查看；支持切换为"上下模式"
缩略图导航条底部横排缩略图列表，当前查看的图片高亮；支持滚轮快速滚动；每张右下角标记状态（未审核/已确认/需重处理）
"确认" 按钮当前图片标记为已确认；快捷键 Enter；确认后自动跳到下一张；已确认图片底部导航标记绿色 ✓
"需重处理" 按钮当前图片标记为需重处理；快捷键 R；标记后底部导航标记橙色 ⚠️
放大/缩放滚轮缩放 + 拖拽平移；双击重置为 100%
"全部导出" 按钮导出所有已确认图片；未确认的图片弹窗提醒"还有 N 张未确认，是否跳过？"
"重处理已标记" 按钮将所有标记"需重处理"的图片送回批量处理页，并保留之前参数
快捷键提示页面底部或浮动显示：← → 翻页 | Enter 确认 | R 重处理 | F 全屏 | Esc 退出

状态与交互：

空状态（无处理后）： 不应当出现此页面，需先完成一次批量处理。
全部确认后： 浮动提示"所有图片已确认，可导出" + 导出按钮放大。
断点续审： 上次审查进度保存在本地（最后查看的图片索引），再次进入时提示"从上次位置继续？"
对比滑条交互细节： 中间分割线跟随鼠标；分割线两侧显示标签"处理前" / "处理后"；鼠标移出对比区域后分割线停留在最后位置。

2.7.5 处理报告（Report）

维度说明
页面目的给出本次批量处理的详细数据报告，支持导出
进入方式处理完成后自动跳转、或从仪表盘最近任务列表进入

核心功能：

  1. 任务摘要 — 处理时间、图片总数、目标人员、美颜参数、消耗时长
  2. 处理统计 — 成功/失败/未检测到目标的图片数量 + 环形图可视化
  3. 详细列表 — 每张图片的处理结果（文件名、处理状态、检测到的人脸数、匹配到目标数、处理耗时）
  4. 失败原因归类 — 按错误类型分组展示（未检测到人脸 / 识别失败 / 处理异常）
  5. 报告导出 — 导出为 JSON / CSV / PDF

UI 元素清单：

元素交互要求
任务摘要区卡片式布局：任务名称 + 时间戳 + 目标人员 Tag 列表 + 参数标签
统计环形图Doughnut Chart，成功(绿) / 失败(红) / 未检测到(灰) 三色；中间显示成功率百分比
详细结果表格可排序表格：文件名 | 人脸数 | 匹配目标数 | 处理耗时(ms) | 状态(成功/失败) | 失败原因 | 操作（预览跳转）
失败归类面板折叠面板（Accordion），按错误类型展开查看具体图片列表
"导出报告" 按钮Dropdown Button，下拉选择 JSON / CSV / PDF
"返回审阅" 按钮Secondary Button，跳回预览审阅页
图片缩略图预览表格中文件名旁显示 40px 微型缩略图；鼠标悬停放大为 120px Popover

状态与交互：

空错误（全部成功）： 环形图全绿，不显示失败归类面板。
有失败项： 失败归类面板自动展开第一个分类；表格中失败行红色高亮。
导出格式差异： JSON 包含完整数据、CSV 适合 Excel 打开、PDF 适合打印/分享。
对比跳转： 点击表格中任意行可直接跳转到预览审阅页的对应图片。

2.8 全局交互设计规范

维度规范要求
全局导航固定在顶部，包含应用名称 + Logo + 5 个页面入口 + 当前页高亮指示器；窄屏时折叠为汉堡菜单
响应式布局桌面端全宽展示；平板端（<1024px）侧边栏自动折叠；手机端（<640px）转为单列堆叠布局
主题模式支持亮色/暗色切换，默认跟随系统 (prefers-color-scheme)

3.4 核心磨皮算法方案详述

方案 A：表面模糊（Surface Blur）— 推荐

模拟 Photoshop 的表面模糊滤镜，在保留边缘的同时平滑皮肤区域：

1. 计算每个像素邻域的色差权重

2. 对色差小于阈值的像素做高斯加权平均（平滑皮肤纹理）

3. 对色差大于阈值的像素保留原值（保护边缘如眼睛/眉毛/嘴唇）

4. 将模糊结果与 Mask 相乘，仅作用于皮肤区域

5. 可选的：叠加高频细节纹理保持真实感

方案 B：双边滤波 + 高频叠加（快速 MVP）

1. 对全图应用双边滤波（bilateral filter），得到平滑层

2. 原图 - 平滑层 = 高频细节层（毛孔/纹理）

3. 以一定比例（如 30%~50%）衰减高频细节层

4. 平滑层 + 衰减后的高频层 = 磨皮结果

5. 通过 Mask 只应用到目标人员皮肤区域

3.5 命令行接口设计（MVP）

# 注册目标人员 python beautify.py register --name "张三" --photos ./refs/zhangsan/*.jpg # 注册多个人员 python beautify.py register --name "张三" --photos ./refs/zhangsan/*.jpg \                --name "李四" --photos ./refs/lisi/*.jpg # 批量处理 python beautify.py process --input ./photos/ --output ./output/ \                 --targets "张三" "李四" \                 --strength medium \                 --report ./report.json

3.6 关键技术决策说明

为什么选择 MediaPipe 而非深度学习方案？
MediaPipe 在 CPU 上即可达到实时速度，无需 GPU，部署成本最低。对于 MVP 阶段，其检测精度（~99%+ 正脸）完全满足要求。后续可切换 RetinaFace 提升精度。
为什么选择表面模糊而非深度学习方法做磨皮？
MVP 阶段应优先保证可交付。表面模糊算法无需模型训练、无 GPU 依赖、参数直观（半径 + 阈值）、效果可控。DNN 磨皮作为 P2 后期迭代。
多人脸处理策略
单张图片中：对所有检测到的人脸进行识别 → 匹配目标人员 → 仅匹配成功的人脸做皮肤分割 + 磨皮 → 非目标人脸不做任何处理。这样可以保证同一张图中不同人员的差异化处理。

四、文件与文件夹排序规范

涵盖批量处理工作台中导入文件列表和预览审阅页缩略图导航条的排序规则，以及导出时目录结构的保持策略。

4.1 导入文件列表排序

维度详细规范
默认排序按文件名升序（自然序 Natural Sort）排列。自然序保证 img2.jpg 排在 img10.jpg 之前，而非字典序的 img10 → img2。
支持的排序方式用户可通过列表表头点击切换排序：
• 文件名升/降（自然序）
• 文件大小升/降
• 文件修改日期升/降
• 文件类型/扩展名分组（.jpg / .png / .webp）
排序操作点击表头切换排序 → 箭头指示方向（▲ 升序 / ▼ 降序）；首次点击升序，再次点击降序，第三次取消排序回到默认。当前排序列表头背景色微加深。
排序持久化用户选择的排序方式和排序方向跨会话保留（存入 localStorage），下次导入新图片列表时沿用上次的排序偏好，直到用户主动更改。

4.2 文件夹导入与结构展示

维度详细规范
导入方式支持两种导入方式，优先推荐选择文件夹（保持目录结构）：
文件夹导入• 选择文件夹后，递归扫描所有子目录中的图片文件
• 导入列表中每行增加"相对路径"列，显示如 2025/group/ 的路径前缀
• 支持按源文件夹分组展示（Group by folder），同一文件夹的图片连续排列，分组间有分隔线/分组标题
• 分组标题显示文件夹名和该文件夹图片数量（如 "2025/group/ (12 张)"）
• 分组可折叠/展开
拖拽/多文件导入• 用户从不同位置拖入多张图片时，文件平铺展示，不保留来源路径信息
• 文件列表头部显示提示："来自 N 个不同位置"
混合导入处理• 如果用户同时拖拽了文件又选择了文件夹，所有文件合并展示
• 有文件夹来源的图片显示相对路径，无来源的图片路径列显示 "-"
目录结构保持• 导出时默认完整保持导入时的目录结构
• 文件夹导入→导出时子目录结构不变
• 散文件拖入→导出时统一放在输出根目录
• 用户可在导出前预览目录结构树
重复文件检测• 依据 文件名 + 文件大小 + 最后修改时间 三元组判断重复
• 重复文件自动跳过，列表底部 Toast "已跳过 N 个重复文件"
• 不同文件夹下的同名文件视为不同文件（因路径不同）
文件过滤• 默认只显示支持的图片格式（jpg/jpeg/png/webp）
• 列表中提供"显示全部/仅图片"切换按钮
• 非图片文件（如 .txt .raw）在列表中隐藏，不影响处理

4.3 预览审阅页排序

维度详细规范
缩略图导航条排序默认与导入时的文件列表顺序一致，保证用户在导入列表和预览页之间看到的是同一顺序。
排序切换预览页底部提供排序切换下拉菜单：
• 按文件名（与导入列表一致）
• 按处理状态（待确认 → 已确认 → 需重处理）
• 按人脸检测数（多人脸优先）
• 手动排序（保持用户拖拽重排的结果）
手动拖拽排序• 支持鼠标拖拽缩略图调整审阅顺序
• 拖拽时其他缩略图自动让位，动画过渡
• 手动排序优先级最高，切换其他排序方式后丢失手动顺序

4.4 空状态与边界情况

无图片文件夹： 选择的文件夹中没有任何支持的图片格式 → 弹窗提示"所选文件夹中未找到图片文件（支持 jpg/png/webp）"。
大量文件（>1,000 张）： 导入时弹出进度条"正在扫描文件…" + 分批渲染列表（Virtual Scroll），避免页面卡顿。
权限不足： 无法读取某些子文件夹 → 跳过该文件夹 + 在报告中标明"N 个文件夹因权限跳过"。
文件名编码异常： 无法解析的文件名用十六进制 hash 代替显示，不阻断整体导入。

5.2 存储策略选择依据

存储方式适用场景容量限制优点缺点
localStorage轻量 KV 配置（布尔值、数字、短字符串）~5MB同步 API，简单可靠只能存字符串，不适合复杂结构和大数据
sessionStorage临时会话状态（无需跨标签页保留）~5MB标签页关闭自动清理不跨会话
IndexedDB结构化数据、Blob 文件、大体积对象~GB 级异步 API，支持索引和事务，存储空间大API 复杂，需封装
文件系统（Electron/PWA）Face Embedding 模型缓存、导出配置到本地文件无限制用户可见、可迁移需文件系统权限

5.3 配置读写规则

规则说明
写入时机实时写入。用户更改任何配置项后立即持久化（非"保存按钮"模式）。例如：拖动磨皮滑杆时，松开鼠标即写入 localStorage。
读取时机应用启动时一次性加载全局配置到内存；运行时通过内存读写，写入时同步持久化。
写入节流高频变化的配置（如滑杆拖动、窗口宽度）做 300ms 防抖写入，避免频繁 I/O。
读写失败处理写入失败 → 静默重试一次 → 失败则 console.warn 不阻塞用户操作；读取失败 → 使用内置默认值，不影响应用启动。
数据迁移配置结构可能随版本更新变化。在读取时检查 schema 版本号（存储在 localStorage / IndexedDB 的版本字段），不匹配时执行迁移函数或丢弃旧数据回退到默认值。
防篡改localStorage 中的配置不做防篡改（用户自行修改不导致功能性错误）；IndexedDB 中的人员数据不暴露直接编辑入口，通过应用内表单修改。

5.4 任务参数回记忆（核心体验）

场景：用户昨天处理了 100 张图，今天要处理另一批图片，目标人员相同、参数类似。

1 用户进入批量处理页 → 导入新图片

2 目标人员选择器自动勾选上次使用的目标人员（右上角显示"沿用上次"标签）

3 磨皮强度滑杆定位到上次使用的值（滑杆下方显示小字"上次：中度 | 边缘保护 70%"）

4 用户可直接点击"开始处理"，无需重新配置

5 如果用户主动调整参数，则更新记忆为新值

记忆项存储方式保留策略
上次选中的目标人员 ID 列表localStorage JSON 数组仅保留最近一次任务，新任务覆盖旧任务
上次使用的磨皮参数（强度 + 高级参数）localStorage 对象同上，覆盖式
上次导出路径localStorage 字符串永久保留，直到用户更改
上次选择的排序方式localStorage 对象永久保留
上次导入方式（文件夹/拖拽）localStorage（仅记录偏好）跨会话，用于默认展示文件分组方式

5.5 预设（Preset）管理

维度详细规范
什么是预设用户保存的一组命名的完整参数配置，包含：磨皮强度 + 边缘保护 + 细节保留 + 目标人员列表（可选）。可用于在不同场景间快速切换。
预设交互• 在批量处理页参数区旁增加"预设"按钮 → 打开预设管理 Drawer
• Drawer 中展示预设列表（名称 + 参数摘要）
• "保存当前为预设" 按钮 → 弹出命名输入框 → 保存到 IndexedDB
• 点击预设 → 立即应用参数到当前配置
• "管理预设" → 编辑名称 / 删除 / 导出为 JSON 文件
内置预设• 系统出厂预设："轻度磨皮"（强度 30, 边缘 90, 细节 80）、"中度磨皮"（强度 55, 边缘 75, 细节 60）、"重度磨皮"（强度 80, 边缘 60, 细节 40）
• 内置预设不可删除，但可基于内置预设保存为用户自定义预设
预设导入/导出• 导出预设为 JSON 文件（包含版本号、创建时间、参数）
• 导入 JSON 预设文件，导入时校验版本兼容性
• 支持分享预设文件给其他用户
预设存储IndexedDB 的 presets 表，每条记录包含：id, name, params (JSON), isBuiltin, createdAt, updatedAt, version

5.6 会话恢复（Crash Recovery）

场景恢复行为
页面刷新（未完成处理）• 刷新后进入仪表盘，右上角出现橙色横幅："有未完成的批量处理任务，是否恢复？"
• 点击"恢复" → 跳转到批量处理页，进度和数据完全恢复（包括已处理的图片）
• 点击"放弃" → 清除 IndexedDB 中的任务状态数据
浏览器崩溃• IndexedDB 中的数据不会丢失
• 下次打开应用时同样显示恢复横幅
• 如果状态数据损坏 → 横幅显示"上次任务状态异常，已自动清理"
处理过程中关闭标签页• 后台处理线程（Web Worker）通知 onbeforeunload 事件
• 在 beforeunload 中提示"处理尚未完成，确定离开？"
• 用户确认离开 → 状态持久化到 IndexedDB
并发的浏览标签页• 同一应用在多个标签页打开时，以最后一个写入 IndexedDB 的数据为准
• 不鼓励多标签页同时使用，检测到多标签页时提示"该应用已在其他标签页打开"
配置数据损坏• localStorage 单字段损坏 → 丢弃该字段，使用默认值
• IndexedDB 数据库损坏 → 重建空数据库，不阻塞启动

批量处理速度太慢（CPU 模式）低中多线程优化；优先使用轻量模型；GPU 加速作为 P2
非目标人脸误识别为目标低高提高相似度阈值；输出中间结果供人工审核
同一人跨图片肤色不一致导致磨皮效果不统一低低统一使用绝对参数而非相对参数
